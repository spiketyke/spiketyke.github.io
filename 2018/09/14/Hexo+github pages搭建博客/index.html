Hexo+github Pages搭建博客

关于Github

Github的优点

·Github是基于git实现的代码托管
·Github免费

什么是Github Pages

·Github Pages是用户编写的、托管在Github上的静态网页
·可以绑定你的域名
·使用Github Pages可以为你提供一个免费的服务器，免去自己搭建服务器和写数据库的麻烦

安装Git & Node

Node.js:一个能够在服务器端运行JavaScript的开放源代码、跨平台JavaScript运行环境
Git：分布式版本控制软件

通过homebrew安装：

brew install git
brew install node

安装Hexo

npm install hexo-cli -g(安装hexo)
npm install hexo-deployer-git --save(安装hexo部署到git page的deployer)

初始化Hexo配置

hexo init（将会在指定文件夹中新建所需文件）

本地查看效果

hexo g
hexo s

登录localhost:4000，即可看到本地效果

将博客部署到Github Pages上

注册Github账户

create a new repository

配置SSH密钥

配置Github的SSH密钥可以让本地git项目与远程的github建立联系，在本地写了代码之后直接通过git操作就可以实现本地代码库与Github代码库同步。

看本地是否存在SSH密钥

cd ~/. ssh

若不存在.ssh目录进行第二步操作，否则本机已存在ssh公钥和私钥，进行第三步

创建一对新的SSH密钥

ssh-keygen -t rsa -C "your_email@example.com"
#将按照你提供的邮箱地址创建一对密钥
Gennerating public/private rsa key pair.
Enter file in which to save the key (/c/Users/you/.ssh/id_rsa): [Press enter]

直接回车，则将密钥按默认文件进行存储。

接下来，根据提示输入密码和确认密码(在输密码的地方可以直接回车，省得日后同步代码时频繁输密码)。相关提示如下：

Enter passphrase (empty for no passphrase): [Type a passphrase]
Enter same passphrase again: [Type passphrase again]

输入完成之后，屏幕显示如下信息：

Your identification has been saved in /c/Users/you/.ssh/id_rsa.
Your public key has been saved in /c/Users/you/.ssh/id_rsa.pub.
The key fingerprint is:
01:0f:f4:3b:ca:85:d6:17:a1:7d:f0:68:9d:f0:a2:db your_email@example.com

在Github账户中添加公钥

运行如下命令，将公钥的内容复制到粘贴板中

clip < ~/.ssh/id_rsa.pub

如果无法复制可进入.ssh文件夹，打开id_rsa.pub，复制其中内容

然后：

      1、登录Github，进入你的Account Settings.

      2、选择SSH Keys

      3、粘贴密钥，添加即可

设置用户信息

现在已经可以通过SSH链接到Github，还有一些个人信息需要完善。Git会根据用户的名字和邮箱来记录提交，Github也是用这些信息来做权限的处理。

git config –global user.name “ryanlijianchang”//用户名
git config –global user.email “liji.anchang@163.com“//填写自己的邮箱

将本地的Hexo文件更新到Github的库中

登录Github打开自己的项目 username.github.io

点击Use SSH
复制地址

打开一开始创建的Hexo文件夹（如K\blog）,打开_config.yml文件在配置文件里作如下修改并保存

在hexo文件下执行

hexo g -d

访问博客

在浏览器输入GIthub pages为我们生成的外链(如：spiketyke.github.io),便可通过这个地址访问你自己的博客了

优化博客

进入Hexo的官网主题选择喜欢的主题

鼠标停留图片上点击进入预览页面，点击标题进入github页面（包含安装方法）

安装主题

git clone https://github.com/your_theme_url

等待下载完成

修改Hexo配置文件

打开Hexo文件夹下的配置文件_comfig.yml
修改参数为：theme: your_theme_name(注意冒号后有空格)

部署主题，本地查看效果

进入Hexo目录
hexo g
hexo s

打开浏览器输入localhost:4000，即可看见主题已更换

将主题部署到Github上

hexo clean（有时因为缓存问题，服务器更新不了主题进行此操作）
hexo g -d

以next主题为例，进入blog/themes/next目录，打开_config.yml文件，command+F，查找livere

默认livere_uid是被注释掉的，将#删除，至于livere_uid到来必力获得

红线所画处，不带引号

添加访客数、浏览量统计功能

同上，command+F，查找footer

enable默认为false，更改为true即可

写博文

用hexo发表新文章

hexo n "title"

执行此命令后会在blog\source_posts中生成title.md文件，用编辑器打开编写即可

亦可在source_posts文件夹下直接新建一个md文件

用Markdown写文章

Hexo支持Markdown语法，用支持Markdown语法的编辑器（如MacDown，支持实时预览）进行文章编译，保存后执行hexo g -d 推送到Github仓库即可

Markdown是什么？

Markdown 是一种轻量级的「标记语言」，它的优点很多，目前也被越来越多的写作爱好者，撰稿者广泛使用。看到这里请不要被「标记」、「语言」所迷惑，Markdown 的语法十分简单。常用的标记符号也不超过十个，这种相对于更为复杂的HTML 标记语言来说，Markdown 可谓是十分轻量的，学习成本也不需要太多，且一旦熟悉这种语法规则，会有一劳永逸的效果。

Markdown优点

  • 专注你的文字内容而不是排版样式。
  • 轻松的导出 HTML、PDF 和本身的 .md 文件。
  • 纯文本内容，兼容所有的文本编辑器与字处理软件。
  • 可读，直观。适合所有人的写作语言。

将自己的域名关联到Github Pages上

购买域名

我选择的是腾讯云，后悔没有购买一元（首年）.club域名而选择了.cn，.cn续费（￥35）比.club续费（￥138）便宜

配置CNAME文件

在blog\source文件夹下创建CNAME（文本编辑器新建文本，无后缀。创建此文件时建议用windows，在Mac下为Unix可执行文件），内容为你的域名地址，域名成功备案后便可访问十分方便。
提交 hexo g -d
